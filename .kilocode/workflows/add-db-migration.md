# Workflow — Make a DB schema change (Alembic)

Use this when changes affect persistent data structures.

## 1) Make the model/schema change

- Update SQLAlchemy models under [`backend/db/`](backend/db/:1) (if applicable).

## 2) Create a migration

- Create a new Alembic revision under [`backend/alembic/versions/`](backend/alembic/versions/:1).
- Prefer autogenerated migrations when possible.

## 3) Review the migration carefully

- Ensure it’s safe for existing data.
- Add defaults/backfills if needed.

## 4) Apply + test

- Apply migrations in dev.
- Run backend tests under [`backend/tests/`](backend/tests/:1).
